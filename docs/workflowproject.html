Project workflow.

We need to answer two question when we start real world project

  1. In our analysis, what will we save for future reference(real)

  2. Where do we save the analysis?

8.1 What is real?

R script is what we will be saving for future reference, all values at the environmental pane will go after our analysis.

With your R scripts (and your data files), you can recreate the environment. That is why, you should always tell Rstudio not to preserve your workspace between sessions. This approach will save you long-term agony because it forces you to capture all important interactions in your code.

Short-cut to captured the important parts of your code in the editor:

Cmd/Ctrl + Shift + F10 to restart RStudio. or use .rs.restartR() to restart s r. My case, command+ shift + ) restart R. Cmd/Ctrl + Shift + S to rerun the current scrip

8.2 Where does your analysis live?

Your analysis should live inside project working directory.

To see current working directory at console use: getwd(): To set working directory (not recommended approach) use:setwd("/path/to/my/CoolProject")

8.3 Paths and directories

Absoulte path also referred to as file path and full path, contains the root directory and all other subdirectories where a file or folder is contained.

There are two basic styles of paths: Mac/Linux and Windows. Three basic ways they differ are:

  1. Always Use forward slashes style

Mac and Linux uses slashes (e.g. plots/diamonds.pdf) and Windows uses backslashes (e.g. plots.pdf). However, backslashes mean something special to R, and to get a single backslash in the path, you need to type two backslashes!. Therefore, it is recommended to the Linux/Mac style with forward slashes.

  1. Never use absolute path, they prevent sharing

In windows, absolute paths starts with a drive letter (e.g. C:) or two backslashes (e.g. \servername)

In MAC/Linxu, absolute paths starts with a slash “/” (e.g. /users/hadley)

Learn more about absolute and relative path here

  1. __ ~ shortcut in MAC/Linux vs Window__

Another difference is the place that ~ points to. ~ is a convenient shortcut to home directory in MAC/Linux while Windows doesn’t have the notion of a home directory, so it point to your documents directory.
